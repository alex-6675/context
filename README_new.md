# Мои контакты и записки

## ЗАПИСКА ДЛЯ НЕЙРОАНАЛИЗАТОРА

**ОБЯЗАТЕЛЕН СЕРВИС СЛИЯНИЯ**

ВАЖНО: Одна персона существует во всех сетях как один профиль. То есть ЭТО одна персона - один профиль.

Карточка профиля должна иметь поля реальных контактов:
- Адреса
- Телефоны
- Должности
- Локации (географические данные из ОК, VK и других соцсетей)

**МОДИФИКАЦИЯ ДЛЯ РЕАЛИЗАЦИИ:**

1. **Сервис слияния профилей** - необходимо разработать алгоритм идентификации одной персоны across всех социальных сетей
2. **Модуль импорта в Я-акк** - требуется реализовать функционал экспорта данных в Яндекс аккаунт потребителя данного сервиса
   - Интеграция через кнопку "сохранить на диск" после верификации пользователя
3. **Универсальная карточка профиля** - создать структуру данных, объединяющую информацию из всех сетей
4. **Заполнение карточки по открытым данным** - разработать механизм автоматического заполнения карточки на основе публичной информации из соцсетей
5. **Активация скрипта изъятия данных** - реализовать функционал для извлечения данных из профилей пользователей в соцсетях

**ТРЕБОВАНИЯ К РЕШЕНИЮ:**
- Быстродействие: минимальное время обработки данных
- Стабильность: надежная работа при больших объемах данных
- Масштабируемость: возможность расширения на новые социальные сети
- Безопасность: защита персональных данных пользователя
- Интеллектуальный анализ: использование ИИ для распознавания и сопоставления данных

**АНАЛИЗ ГОТОВЫХ РЕШЕНИЙ:**
Необходимо исследовать существующие успешные быстрые и стабильные решения для выполнения задач Данного проекта и предложить модульную инсталляцию существующего. 

## ВНИМАНИЕ: ИДЕТ РАЗРАБОТКА

Индивидуальный помощник для ведения записок о взаимодействиях с клиентами и контактами в социальных сетях, оформленный как расширение для браузера (webextension), поддерживаются десктопные браузеры, созданные на базе Chrome.

Перечень поддерживаемых социальных сетей расширяется, на данный момент поддерживаются следующие соцсети:
- Вконтакте (vk.com)
- OK (ok.ru)  
- Rutube (rutube.ru)
- Живой Журнал (livejournal.com) - низкий приоритет
- Хабр (habr.com) - низкий приоритет

### Основная идея

Инструмент сохранения взаимодейстий

Расширение "Мои контакты и записки" помогает малому бизнесу, самозанятым и фрилансерам вести учет взаимодействий с клиентами и потенциальными заказчиками. Данные о событиях хранятся локально, во встроенной БД вашего браузера – поэтому недоступны никому, кроме вас, отредактировать или удалить сведения о том ли ином событии можете только вы. (концепция под вопросом - нужна выгрузка в акк Яндекса)

### Примеры использования

#### Пример 1: Цветочный бизнес
- Уточнение наличия сортов (например, "Фердинант - поздняя" груша)
- Отслеживание интересов клиентов (например, "мята не интересна")
- Маркировка клиентов для персонализированных предложений

#### Пример 2: Блокировка нерелевантных контактов
- Клиенты, которые не заинтересованы в ваших услугах
- Возможность скрыть нерелевантные комментарии
- Визуальное выделение заблокированных контактов

### Принципы работы

#### Принцип "Покупатель всегда прав"
- Никакого спама или навязывания услуг
- Уважение к выбору клиента
- Возможность отмечать нерелевантные контакты без агрессивной блокировки

#### Система маркеров и заметок
- Создание заметок о предпочтениях клиентов
- Маркировка контактов для персонализированных предложений
- Отслеживание истории взаимодействий

#### Гибкая система фильтрации
- Визуальное выделение заблокированных контактов (бледный цвет)
- Возможность просмотра, но не активного взаимодействия
- Отдельное действие для снятия блокировки

### Основные компоненты

#### 1. Background Script (background.js)
- Управление базой данных IndexedDB
- Обработка сообщений от контент-скриптов
- Управление состоянием расширения

#### 2. Content Scripts
- Наблюдение за изменениями DOM (mutation.js)
- Анализ содержимого страниц социальных сетей
- Выделение пользователей и событий
- Отображение UI элементов (меню, бейджи)

#### 3. Social Network Adapters
- Специфическая логика для каждой социальной сети
- Идентификация пользователей и событий
- Извлечение данных (даты, тексты, ссылки)

#### 4. Storage Layer
- Работа с IndexedDB
- Хранение заметок о клиентах
- Хранение истории взаимодействий

#### 5. UI Components
- Popup окно
- Страница настроек
- Диалоги событий и информации о пользователях

#### 6. Utility Libraries
- Работа с датами
- Обработка строк
- Экспорт/импорт данных

## Избыточные блоки и дублирование

### 1. Дублирование функций работы с DOM
- Функции типа `getParentElementBelobgsToClass`, `getIndirectChildElementBelongsToClass` дублируются в нескольких файлах
- Могут быть вынесены в общий utility модуль

### 2. Дублирование функций обработки дат
- Функции `parceDateFromRuLocale` находятся и в `contextlib_mod.js`, и в `contextlib.js`
- Нарушение принципа единственной ответственности

### 3. Избыточные глобальные переменные
- Множество глобальных переменных (`gUsersCache`, `gRanksParams`, `ActiveZones`)
- Могут быть заменены на централизованное управление состоянием

### 4. Дублирование логики обработки сообщений
- Похожая структура обработки сообщений в разных частях приложения
- Может быть унифицирована

### 5. Избыточная сложность в обработке статусов
- Сложная система идентификации и обновления статусов пользователей
- Может быть упрощена с использованием современных подходов к управлению состоянием

### 6. Дублирование UI логики
- Похожие диалоги и обработчики в разных частях приложения
- Может быть унифицировано
