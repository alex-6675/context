# Список дополнительных инструментов для проекта «Мои контакты и записки»

Для эффективной разработки, тестирования, отладки и поддержки расширения рекомендуется использовать следующие инструменты.

## 1. Интегрированная среда разработки (IDE)

### Visual Studio Code (рекомендуется)
- **Обязательные расширения**:
  1. **ESLint** — линтинг JavaScript/TypeScript кода.
  2. **Prettier** — автоматическое форматирование кода.
  3. **Chrome Debugger** — отладка расширений Chrome прямо из VS Code.
  4. **GitLens** — улучшенная работа с Git.
  5. **Live Server** — быстрый просмотр HTML/CSS (для UI компонентов).
  6. **Code Spell Checker** — проверка орфографии (включая русский язык).
  7. **JSDoc Tag Completion** — автодополнение JSDoc комментариев.
  8. **Import Cost** — отображение размера импортируемых модулей.
  9. **TODO Highlight** — подсветка TODO, FIXME комментариев.
  10. **Error Lens** — отображение ошибок и предупреждений прямо в коде.

- **Настройки workspace**:
  - Включить форматирование при сохранении.
  - Установить ESLint как линтер по умолчанию.
  - Настроить Prettier с правилами: одинарные кавычки, отступ 2 пробела, без точки с запятой (если соответствует стилю проекта).

## 2. Система сборки и менеджер пакетов

### Node.js (версия 18+)
- **Необходимость**: для запуска скриптов сборки, тестирования, линтинга.
- **Менеджер пакетов**: npm или yarn (рекомендуется npm, встроенный в Node.js).

### Webpack или Parcel
- **Цель**: сборка расширения в production-режиме (минификация, объединение файлов, tree shaking).
- **Конфигурация**: отдельные конфиги для development и production.

### Скрипты package.json
Примерный набор скриптов:
```json
{
  "scripts": {
    "dev": "webpack --mode development --watch",
    "build": "webpack --mode production",
    "lint": "eslint .",
    "lint:fix": "eslint . --fix",
    "test": "jest",
    "test:watch": "jest --watch",
    "pack": "crx pack ./dist -o ./build/extension.crx",
    "zip": "bestzip extension.zip ./dist/*"
  }
}
```

## 3. Инструменты для тестирования

### Jest
- **Для модульного и интеграционного тестирования**.
- **Конфигурация**: поддержка ES6 модулей, mock для Chrome API.
- **Дополнительные пакеты**:
  - `jest-chrome` — моки для chrome.* API.
  - `@testing-library/dom` — тестирование DOM.
  - `jest-environment-jsdom` — среда для тестирования UI.

### Puppeteer или Playwright
- **Для end-to-end тестирования** расширения в реальном браузере.
- **Автоматизация**: открытие Chrome с установленным расширением, симуляция действий пользователя.

### Chrome DevTools
- **Инструменты разработчика** для отладки background scripts, content scripts, popup.
- **Инспектор Network** для мониторинга запросов к API.
- **Application tab** для просмотра IndexedDB, localStorage.

## 4. Инструменты для анализа кода и безопасности

### ESLint
- **Конфигурация**: `eslint-config-airbnb-base` или собственная.
- **Плагины**:
  - `eslint-plugin-security` — обнаружение потенциальных уязвимостей.
  - `eslint-plugin-no-unsanitized` — проверка на использование innerHTML.
  - `eslint-plugin-compat` — проверка совместимости с браузерами.

### SonarQube или SonarCloud
- **Статический анализ** кода на качество, покрытие тестами, security hotspots.

### OWASP ZAP (Zed Attack Proxy)
- **Динамический анализ безопасности** расширения (XSS, CSRF, инъекции).

## 5. Инструменты для работы с Chrome расширениями

### chrome-extension-cli
- **Генератор шаблонов** для быстрого создания структуры расширения.

### crx (Chrome Extension)
- **Утилита для упаковки** расширения в .crx файл.

### web-ext
- **Инструмент от Mozilla**, но работает с Chrome. Позволяет запускать расширение в изолированном окружении, автоматически перезагружать при изменениях.

### Extension Reloader
- **Расширение для Chrome**, которое автоматически перезагружает ваше расширение при изменении файлов (удобно при разработке).

## 6. Инструменты для документации

### JSDoc
- **Генерация документации** из комментариев в коде.
- **Тема**: можно использовать `better-docs` для красивого вывода.

### Docusaurus или VuePress
- **Для создания полноценной документации** проекта (пользовательской и технической).

### Draw.io / Excalidraw
- **Для рисования диаграмм** архитектуры, workflow, UML.

## 7. Инструменты для CI/CD

### GitHub Actions
- **Автоматизация** сборки, тестирования, линтинга при каждом пуше.
- **Workflow пример**:
  - Установка Node.js, Chrome.
  - Запуск `npm run lint`.
  - Запуск `npm test`.
  - Сборка расширения и загрузка артефактов.

### Chrome Web Store API
- **Для автоматической публикации** обновлений в Chrome Web Store.

## 8. Инструменты для управления проектом

### Trello / Jira / Asana
- **Трекинг задач** в соответствии с регламентом работ.

### Notion / Confluence
- **Ведение документации** проекта, meeting notes, решения.

### Figma / Adobe XD
- **Дизайн интерфейса** расширения (popup, options page, диалоги).

## 9. Инструменты для анализа данных

### JSON Server
- **Моковый REST API** для тестирования интеграции с Яндекс (если нужно).

### Faker.js
- **Генерация тестовых данных** для заполнения карточек контактов.

### Excel / Google Sheets
- **Анализ экспортированных данных** (CSV/JSON) для проверки корректности.

## 10. Прочие утилиты

### nvm (Node Version Manager)
- **Управление версиями Node.js** на одной машине.

### Git
- **Обязательно** с настроенными алиасами, хуками pre-commit (например, запуск линтера).

### Docker
- **Контейнеризация** окружения для воспроизводимости сборки и тестов.

### WSL2 (если разработка ведётся на Windows)
- **Для запуска Linux-инструментов** (например, некоторых CLI утилит).

## 11. Рекомендуемые онлайн-сервисы

- **BrowserStack** — тестирование на разных версиях Chrome и ОС.
- **Sentry** — мониторинг ошибок в production.
- **Google Analytics** (опционально) — сбор анонимной статистики использования расширения (только с согласия пользователя).

## 12. Заключение

Данный набор инструментов покрывает все этапы разработки: от написания кода до публикации и поддержки. Рекомендуется установить и настроить их перед началом активной работы над проектом.

*Примечание: некоторые инструменты могут быть избыточны для маленькой команды; выбирайте по необходимости.*
